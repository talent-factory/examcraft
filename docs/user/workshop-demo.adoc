== Demo Script

=== Opening (2 minutes)

==== Hook
> "Raise your hand if you've ever spent a weekend creating exam questions..."
>
> "What if I told you that same task could be done in 10 minutes?"

==== Problem Statement
> "Today's challenge for educators:
>
> * ⏰ **Time**: Hours spent creating quality questions
> * 🎯 **Consistency**: Maintaining uniform difficulty levels
> * 📚 **Coverage**: Ensuring comprehensive content alignment
> * 🔄 **Updates**: Refreshing questions when materials change"

==== Solution Introduction
> "ExamCraft AI solves this with artificial intelligence that reads your course materials and generates professional-quality exam questions automatically."

=== Live Demonstration (12 minutes)

==== Step 1: Document Upload (3 minutes)

**Presenter Action**: Navigate to dashboard

[source]
----
"Let me show you how simple this is. I'm starting with our Python programming course materials."
----

**Demo Steps**:

1. Open ExamCraft AI dashboard
2. Click "Upload Documents" or drag & drop
3. Select "Python Programming Basics.pdf"
4. Show upload progress
5. Explain processing steps

**Commentary**:

> "Notice how the system automatically:
>
> * Extracts text from the PDF
> * Analyzes the content structure
> * Creates semantic understanding
> * Prepares it for question generation
>
> This usually takes 30-60 seconds depending on document size."

==== Step 2: Question Generation (4 minutes)

**Presenter Action**: Navigate to Generate Questions

[source]
----
"Now comes the magic - generating questions from our content."
----

**Demo Steps**:

1. Select uploaded document
2. Choose topic: "Python variables and data types"
3. Set difficulty: "Medium"
4. Request 3 questions
5. Click "Generate Questions"

**During Generation**:

> "Behind the scenes, the AI is:
>
> * Finding relevant content sections
> * Understanding context and relationships
> * Creating questions at the right difficulty level
> * Generating plausible wrong answers
> * Providing explanations for correct answers"

**Show Results**:

Display generated questions and highlight:

* Question quality and clarity
* Appropriate difficulty level
* Source attribution
* Professional formatting

==== Step 3: Question Review (3 minutes)

**Presenter Action**: Review and edit questions

[source]
----
"The AI does excellent work, but you maintain full control."
----

**Demo Steps**:

1. Review first generated question
2. Show edit capability (change wording)
3. Approve question for use
4. Demonstrate rejection/regeneration

**Commentary**:

> "Key features of the review process:
>
> * **Full Editorial Control**: Change anything you want
> * **Quality Indicators**: Difficulty and learning objective alignment
> * **Source Tracking**: See exactly which content generated each question
> * **Approval Workflow**: Only approved questions enter your question bank"

==== Step 4: Exam Export (2 minutes)

**Presenter Action**: Create and export exam

[source]
----
"Finally, let's create a complete exam."
----

**Demo Steps**:

1. Navigate to Exam Creator
2. Select approved questions
3. Arrange by difficulty/topic
4. Generate PDF preview
5. Show export options

**Commentary**:

> "Export features include:
>
> * **Multiple Formats**: PDF, Word, Moodle XML
> * **Custom Templates**: Your institution's branding
> * **Answer Keys**: Separate or integrated
> * **Metadata**: Learning objectives, time estimates"

== Q&A Preparation

=== Anticipated Questions

==== "How accurate are the AI-generated questions?"

**Answer**:

> "The questions are based directly on your uploaded content, so accuracy is very high. The AI doesn't invent information - it creates questions from what you've provided. Plus, you review every question before use, maintaining full quality control."

==== "What about sensitive or confidential content?"

**Answer**:

> "Security is a priority. Documents are processed locally or in your secure environment. We can deploy on-premises for complete data control. Content is never shared with external parties except the AI service, which has enterprise-grade security."

==== "Can it handle different subject areas?"

**Answer**:

> "Yes! We've tested with subjects ranging from computer science to literature to business. The AI adapts to the content type and maintains appropriate academic language for each discipline."

==== "How much does it cost?"

**Answer**:

> "Costs are primarily the AI processing fees, which average $0.10-0.50 per document depending on size. Compared to faculty time savings, the ROI is typically 50:1 or better."

==== "What about integration with our LMS?"

**Answer**:

> "We export to standard formats including Moodle XML, Canvas QTI, and others. Direct API integration is also possible for seamless workflow integration."

==== "How do we get started?"

**Answer**:

> "We can start with a pilot program using a few of your courses. Setup typically takes 1-2 days, and training takes about an hour. We provide full support during the trial period."

=== Handling Objections

==== "Students will figure out it's AI-generated"

**Response**:

> "The questions are indistinguishable from human-created ones because they're based on your actual content and reviewed by you. The AI is just a tool that speeds up the creation process."

==== "This will replace educators"

**Response**:

> "Absolutely not! This frees educators to focus on what they do best - teaching, mentoring, and curriculum design. It's like having a research assistant for question creation."

==== "AI might make mistakes"

**Response**:

> "That's why we include human review in every workflow. You approve every question. The AI accelerates creation, but human expertise ensures quality."

=== Common Technical Issues

==== Port Conflicts

[source,bash]
----
# Quick resolution
sudo kill -9 $(lsof -t -i:3000)
sudo kill -9 $(lsof -t -i:8000)
./start-dev.sh
----

==== Slow Response

* Blame on "conference WiFi"
* Switch to backup materials
* Focus on Q&A and discussion

==== Claude API Limits

* Show demo mode features
* Use pre-generated examples
* Discuss production scaling
