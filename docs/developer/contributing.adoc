Welcome to the ExamCraft AI contributor community! This guide covers everything you need to know to contribute effectively to the project.

== Getting Started

=== Prerequisites

Before contributing, ensure you have:

* **Development Environment**: Working local setup (see link:quick-start.adoc[Quick Start Guide])
* **Git Knowledge**: Basic understanding of Git workflows
* **Code Editor**: VS Code, IntelliJ, or your preferred editor
* **GitHub Account**: For submitting pull requests

=== Code of Conduct

We are committed to providing a welcoming and inclusive environment for all contributors. Please:

* **Be respectful** in all interactions
* **Focus on constructive feedback** in code reviews
* **Help newcomers** get up to speed
* **Report inappropriate behavior** to maintainers
* **Follow professional standards** in all communications

== Development Workflow

=== 1. Setting Up Your Fork

[source,bash]
----
# Fork the repository on GitHub, then clone your fork
git clone https://github.com/your-username/ExamCraft.git
cd ExamCraft

# Add upstream remote
git remote add upstream https://github.com/original-org/ExamCraft.git

# Verify remotes
git remote -v
----

=== 2. Creating a Feature Branch

[source,bash]
----
# Always start from the latest main branch
git checkout main
git pull upstream main

# Create a feature branch with descriptive name
git checkout -b feature/document-batch-processing
# or
git checkout -b fix/memory-leak-in-vector-service
# or
git checkout -b docs/improve-api-examples
----

**Branch Naming Conventions**:

* `feature/description` - New features
* `fix/description` - Bug fixes
* `docs/description` - Documentation changes
* `refactor/description` - Code refactoring
* `test/description` - Test improvements
* `chore/description` - Maintenance tasks

=== 4. Committing Changes

==== Commit Message Format

Follow the Conventional Commits specification:

[source]
----
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
----

**Examples**:
[source]
----
feat(api): add batch document processing endpoint

Implements bulk upload functionality for processing multiple
documents simultaneously. Includes rate limiting and progress tracking.

Closes #123

fix(frontend): resolve memory leak in document preview

The document preview component was not properly cleaning up
event listeners, causing memory leaks during navigation.

refactor(vector): optimize similarity search algorithm

Improves query performance by 40% using more efficient
indexing strategy and caching frequently accessed vectors.

docs(api): add examples for question generation endpoints

Adds comprehensive examples showing different question types,
difficulty levels, and response formats.
----

**Commit Types**:

* `feat` - New features
* `fix` - Bug fixes
* `docs` - Documentation changes
* `style` - Code style changes (formatting, etc.)
* `refactor` - Code refactoring
* `test` - Adding or modifying tests
* `chore` - Maintenance tasks
* `perf` - Performance improvements
* `ci` - CI/CD changes

**Error Handling**:
[source,python]

# Use custom exceptions
class DocumentProcessingError(Exception):
    """Raised when document processing fails."""

# Proper exception handling
async def process_document(file_path: str) -> ProcessedDocument:
    try:
        content = await extract_content(file_path)
        return ProcessedDocument(content=content)
    except FileNotFoundError:
        raise DocumentProcessingError(f"File not found: {file_path}")
    except Exception as e:
        logger.error(f"Unexpected error processing {file_path}: {e}")
        raise DocumentProcessingError("Processing failed")

**API Integration**:
[source,typescript]
----
// Use proper error types
interface ApiError {
  message: string;
  code: string;
  details?: Record<string, any>;
}

// Service with error handling
export class DocumentService {
  static async upload(file: File): Promise<Document> {
    try {
      const formData = new FormData();
      formData.append('file', file);

      const response = await api.post<Document>('/documents/upload', formData);
      return response.data;
    } catch (error) {
      if (axios.isAxiosError(error)) {
        const apiError: ApiError = error.response?.data || {
          message: 'Upload failed',
          code: 'UPLOAD_ERROR'
        };
        throw new Error(apiError.message);
      }
      throw error;
    }
  }
}
----

=== Database Guidelines

**Migration Files**:
[source,python]
----
# Always include both upgrade and downgrade
def upgrade():
    """Add document metadata column."""
    op.add_column('documents',
        sa.Column('metadata', sa.JSON(), nullable=True)
    )
    op.create_index('idx_documents_metadata', 'documents', ['metadata'])

def downgrade():
    """Remove document metadata column."""
    op.drop_index('idx_documents_metadata')
    op.drop_column('documents', 'metadata')
----

**Model Definitions**:
[source,python]
----
class Document(Base):
    """Document model with proper relationships."""
    __tablename__ = 'documents'

    id = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    filename = Column(String(255), nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow)

    # Relationships
    questions = relationship("Question", back_populates="source_document")

    def __repr__(self) -> str:
        return f"<Document(id={self.id}, filename='{self.filename}')>"
----

## Testing Guidelines

=== Test Structure

**Backend Tests**:
[source,python]
----
# test_document_service.py
import pytest
from unittest.mock import AsyncMock, patch

class TestDocumentService:
    """Test cases for DocumentService."""

    @pytest.fixture
    def document_service(self):
        """Create DocumentService instance for testing."""
        return DocumentService()

    @pytest.mark.asyncio
    async def test_upload_document_success(self, document_service):
        """Test successful document upload."""
        # Arrange
        mock_file = AsyncMock()
        mock_file.filename = "test.pdf"

        # Act
        result = await document_service.upload_document(mock_file, "user123")

        # Assert
        assert result.filename == "test.pdf"
        assert result.user_id == "user123"

    @pytest.mark.asyncio
    async def test_upload_document_invalid_file(self, document_service):
        """Test upload with invalid file type."""
        mock_file = AsyncMock()
        mock_file.filename = "test.exe"

        with pytest.raises(ValidationError):
            await document_service.upload_document(mock_file, "user123")
----

**Frontend Tests**:
[source,typescript]
----
// DocumentUpload.test.tsx
import { render, screen, fireEvent, waitFor } from '@testing-library/react';
import { DocumentUpload } from './DocumentUpload';

describe('DocumentUpload', () => {
  const mockOnUploadComplete = jest.fn();

  beforeEach(() => {
    mockOnUploadComplete.mockClear();
  });

  it('renders upload area', () => {
    render(<DocumentUpload onUploadComplete={mockOnUploadComplete} />);

    expect(screen.getByText(/drag.*drop.*files/i)).toBeInTheDocument();
  });

  it('handles file drop', async () => {
    const mockUpload = jest.spyOn(DocumentService, 'upload')
      .mockResolvedValue({ id: 'doc123', filename: 'test.pdf' });

    render(<DocumentUpload onUploadComplete={mockOnUploadComplete} />);

    const file = new File(['content'], 'test.pdf', { type: 'application/pdf' });
    const dropzone = screen.getByText(/drag.*drop/i);

    fireEvent.drop(dropzone, { dataTransfer: { files: [file] } });

    await waitFor(() => {
      expect(mockUpload).toHaveBeenCalledWith(file);
      expect(mockOnUploadComplete).toHaveBeenCalledWith('doc123');
    });
  });
});
----

== Release Process

=== Version Numbering

We use Semantic Versioning (SemVer):

* **Major** (1.0.0): Breaking changes
* **Minor** (0.1.0): New features, backward compatible
* **Patch** (0.0.1): Bug fixes, backward compatible

=== Release Checklist

**Before Release**:

- [ ] All tests pass
- [ ] Documentation updated
- [ ] Changelog updated
- [ ] Version numbers bumped
- [ ] Breaking changes documented

**Release Steps**:
[source,bash]
----
# Update version
git checkout main
git pull upstream main

# Create release branch
git checkout -b release/v1.2.0

# Update version files
# Update CHANGELOG.md
# Update documentation

# Create release commit
git commit -m "chore: prepare release v1.2.0"

# Create pull request for release
# After approval and merge, tag the release
git tag -a v1.2.0 -m "Release v1.2.0"
git push upstream v1.2.0
----
