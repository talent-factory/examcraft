== 5-Minute Setup

=== Step 1: Clone Repository (30 seconds)

[source,bash]
----
# Clone the repository
git clone <repository-url>
cd ExamCraft

# Verify you're in the right directory
ls -la
# You should see: docker-compose.yml, start-dev.sh, README.md
----

=== Step 2: Environment Configuration (1 minute)

[source,bash]
----
# Copy environment template
cp .env.example .env

# Edit environment file (optional for demo mode)
nano .env
----

**Environment Variables**:
[source,env]
----
# Required for full functionality
CLAUDE_API_KEY=your_claude_api_key_here

# Optional - defaults work for development
DATABASE_URL=postgresql://examcraft:examcraft_dev@postgres:5432/examcraft_dev
REDIS_URL=redis://redis:6379
----

[NOTE]
====
**Demo Mode**: ExamCraft works without Claude API key in demo mode with pre-generated sample questions.
====

=== Step 3: Start Development Environment (3 minutes)

[source,bash]
----
# Start all services
./start-dev.sh

# Or manually with docker-compose
docker-compose up --build -d
----

**What happens during startup**:
1. Downloads and builds Docker images (~2 minutes first time)
2. Starts PostgreSQL, Redis, Backend, and Frontend
3. Runs database migrations
4. Performs health checks

=== Step 4: Verify Installation (30 seconds)

Open these URLs in your browser:

[cols="1,2,3"]
|===
|Service |URL |Expected Result

|**Frontend**
|http://localhost:3000
|ExamCraft AI dashboard loads

|**Backend API**
|http://localhost:8000
|JSON response: `{"status": "ok"}`

|**API Documentation**
|http://localhost:8000/docs
|Interactive Swagger UI
|===

=== Step 5: Test Basic Functionality (30 seconds)

[source,bash]
----
# Test API health
curl http://localhost:8000/health

# Expected response:
# {
#   "status": "healthy",
#   "database": true,
#   "redis": true,
#   "timestamp": "2025-09-23T10:00:00Z"
# }
----

== Common Issues & Solutions

=== Port Already in Use

[source,bash]
----
# Check what's using the ports
lsof -i :3000
lsof -i :8000

# Kill conflicting processes
sudo kill -9 $(lsof -t -i:3000)
sudo kill -9 $(lsof -t -i:8000)

# Restart ExamCraft
./start-dev.sh
----

=== Docker Issues

[source,bash]
----
# Restart Docker Desktop
# Then clean up and restart

# Remove all containers and volumes
docker-compose down -v

# Clean up Docker system
docker system prune -f

# Start fresh
./start-dev.sh
----

=== Services Won't Start

[source,bash]
----
# Check service logs
docker-compose logs backend
docker-compose logs frontend
docker-compose logs postgres

# Common fixes:
# 1. Wait 30 seconds for database initialization
# 2. Restart specific service: docker-compose restart backend
# 3. Rebuild: docker-compose up --build -d
----

=== Database Connection Errors

[source,bash]
----
# Check if PostgreSQL is ready
docker exec examcraft_postgres pg_isready -U examcraft

# Reset database if needed
docker-compose down -v
docker-compose up -d postgres
# Wait 30 seconds
docker-compose up -d
----

== Development Workflow

=== Making Code Changes

**Frontend Development**:
[source,bash]
----
# Frontend auto-reloads on file changes
cd frontend
npm start
# Opens http://localhost:3000 with hot reload
----

**Backend Development**:
[source,bash]
----
# Backend auto-reloads with uvicorn
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
----

=== Running Tests

[source,bash]
----
# Backend tests
cd backend
pytest

# Frontend tests
cd frontend
npm test

# All tests with Docker
docker-compose -f docker-compose.test.yml up --build
----

=== Useful Commands

[source,bash]
----
# View logs
docker-compose logs -f                    # All services
docker-compose logs -f backend            # Backend only
docker-compose logs -f frontend           # Frontend only

# Restart services
docker-compose restart                    # All services
docker-compose restart backend            # Backend only

# Access database
docker exec -it examcraft_postgres psql -U examcraft -d examcraft_dev

# Stop everything
docker-compose down

# Nuclear option: clean everything
docker-compose down -v
docker system prune -f --volumes
----

== Troubleshooting

=== Performance Issues

**Slow Docker on Mac**:
[source,bash]
----
# Increase Docker resources in Docker Desktop:
# - CPU: 4+ cores
# - Memory: 8+ GB
# - Disk image size: 64+ GB
----

**Slow Database**:
[source,bash]
----
# Check if database needs more time to start
docker-compose logs postgres

# Increase shared_buffers in PostgreSQL config
# Add to docker-compose.yml:
# command: postgres -c shared_buffers=256MB
----

=== Network Issues

**API Calls Failing**:
[source,bash]
----
# Check if backend is accessible
curl http://localhost:8000/health

# Check Docker network
docker network ls
docker network inspect examcraft_default
----

**Frontend Can't Reach Backend**:
[source,javascript]
----
// Check proxy configuration in frontend/package.json
"proxy": "http://backend:8000"
----
