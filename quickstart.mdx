## Installation

<Steps>
  <Step title="Repository klonen">
    Klonen Sie das ExamCraft AI Repository von GitHub:

    ```bash
    git clone https://github.com/talent-factory/examcraft.git
    cd examcraft
    ```
  </Step>

  <Step title="Environment-Variablen konfigurieren">
    Kopieren Sie die Beispiel-Konfiguration und passen Sie sie an:

    ```bash
    cp .env.example .env
    ```

    Öffnen Sie `.env` und konfigurieren Sie mindestens:

    ```bash
    # Datenbank-Konfiguration
    POSTGRES_USER=examcraft
    POSTGRES_PASSWORD=your-secure-password
    POSTGRES_DB=examcraft

    # Redis-Konfiguration
    REDIS_PASSWORD=your-redis-password

    # Claude API (optional für Core)
    ANTHROPIC_API_KEY=your-claude-api-key

    # JWT Secret
    JWT_SECRET_KEY=your-jwt-secret-key
    ```

    <Warning>
      Ändern Sie alle Passwörter und Secrets vor dem produktiven Einsatz!
    </Warning>
  </Step>

  <Step title="Services starten">
    Das `start-dev.sh` Script erkennt automatisch, welche Packages verfügbar sind:

    ```bash
    chmod +x start-dev.sh
    ./start-dev.sh
    ```

    Das Script startet automatisch:
    - PostgreSQL Datenbank
    - Redis Cache
    - Backend API (FastAPI)
    - Frontend (React)
    - Qdrant Vector Database (wenn Premium/Enterprise verfügbar)

    <Info>
      Der erste Start kann 2-3 Minuten dauern, während Docker die Images herunterlädt.
    </Info>
  </Step>

  <Step title="Zugriff auf die Anwendung">
    Nach erfolgreichem Start erreichen Sie ExamCraft AI unter:

    - **Frontend**: http://localhost:3000
    - **Backend API**: http://localhost:8000
    - **API Dokumentation**: http://localhost:8000/docs

    <Check>
      Wenn Sie die Login-Seite sehen, war die Installation erfolgreich!
    </Check>
  </Step>
</Steps>

## Deployment-Modi

ExamCraft AI unterstützt verschiedene Deployment-Modi je nach verfügbaren Packages:

<Tabs>
  <Tab title="Core (Open Source)">
    **Community Edition** mit grundlegenden Features

    ```bash
    ./start-dev.sh --core
    ```

    **Verfügbare Features:**
    - ✅ Dokumenten-Upload (5 Dokumente)
    - ✅ Basis-Fragenerstellung (20 Fragen/Monat)
    - ✅ Question Review
    - ✅ User Management
    - ✅ RBAC System

    **Nicht verfügbar:**
    - ❌ RAG Generation
    - ❌ ChatBot
    - ❌ Vector Database
  </Tab>

  <Tab title="Full (Premium + Enterprise)">
    **Komplette Version** mit allen Features

    ```bash
    ./start-dev.sh --full
    ```

    **Zusätzliche Features:**
    - ✅ RAG-basierte Fragenerstellung
    - ✅ Dokument ChatBot
    - ✅ Qdrant Vector Database
    - ✅ Advanced Prompt Management
    - ✅ Semantic Search
    - ✅ Alle Subscription Tiers
  </Tab>

  <Tab title="Auto-Detection">
    **Automatische Erkennung** (empfohlen)

    ```bash
    ./start-dev.sh
    ```

    Das Script erkennt automatisch:
    - Core Package (immer vorhanden)
    - Premium Package (falls initialisiert)
    - Enterprise Package (falls initialisiert)

    Und startet die entsprechenden Services.
  </Tab>
</Tabs>

  <Accordion title="Services überprüfen" icon="list-check">

## Troubleshooting

<AccordionGroup>
  <Accordion title="Services starten nicht" icon="triangle-exclamation">
    **Problem**: Docker Compose schlägt fehl

    **Lösungen:**
    1. Überprüfen Sie, ob Docker läuft: `docker ps`
    2. Prüfen Sie die Logs: `docker compose logs`
    3. Ports bereits belegt? Ändern Sie Ports in `.env`
    4. Environment-Variablen korrekt? Prüfen Sie `.env`
  </Accordion>

  <Accordion title="Frontend lädt nicht" icon="triangle-exclamation">
    **Problem**: Weißer Bildschirm oder Fehler im Browser

    **Lösungen:**
    1. Überprüfen Sie Frontend Logs: `docker compose logs frontend`
    2. Browser-Cache leeren: Strg+Shift+R
    3. Warten Sie 30-60 Sekunden nach dem Start
    4. Überprüfen Sie ob Backend läuft: `curl http://localhost:8000/health`
  </Accordion>

  <Accordion title="Datenbank-Connection Fehler" icon="triangle-exclamation">
    **Problem**: Backend kann nicht mit Datenbank verbinden

    **Lösungen:**
    1. Überprüfen Sie PostgreSQL Container: `docker ps | grep postgres`
    2. Prüfen Sie Credentials in `.env`
    3. Warten Sie 10-15 Sekunden nach dem Start
    4. Logs prüfen: `docker compose logs postgres`
  </Accordion>

  <Accordion title="'start-dev.sh' Permission Denied" icon="triangle-exclamation">
    **Problem**: Script kann nicht ausgeführt werden

    **Lösung:**
    ```bash
    chmod +x start-dev.sh
    ./start-dev.sh
    ```
  </Accordion>
</AccordionGroup>

## Services stoppen

Wenn Sie die Entwicklung beenden möchten:

```bash
./stop-dev.sh
```

Oder manuell:

```bash
docker compose down
```

<Warning>
  Daten bleiben in Docker Volumes erhalten. Zum kompletten Reset verwenden Sie:
  ```bash
  docker compose down -v
  ```
</Warning>
